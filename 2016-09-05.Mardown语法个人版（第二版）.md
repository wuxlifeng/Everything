## Markdown 是什么？

`Markdown` 既不是工具，也不是程序语言，而是一种轻量级的使用纯文本编写的“标记语言”。

Markdown 写作可以直接在 txt 记事本里进行，也可以选择专业编辑器。后者可以实时预览排版效果，方便点选菜单；但 Markdown 语法相对来说非常简单，掌握基本语法后即使在记事本里写作也是完全没有问题的。这样就能摆脱写作环境和条件限制。

## Markdown 基础语法

写作过程中，一般需要这样的排版：
* 标题
* 引用
* 强调（加粗、斜体、下划线、删除线）
* 插入链接、插入图片
* 列表（有序列表、无序列表）
* 表格
* 脚注

### 标题（Headers）

Markdown 可支持 6 个级别的标题（对应于 html 的 `h1` ~ `h6`）。
写法/语法：使用井号 `#` 加`空格`在行首来创建标题。几个井号 `#` 就是几级标题。

```
# 第一级标题
## 第二级标题
### 第三级标题
#### 第四级标题
##### 第五级标题
###### 第六级标题
```

效果如下：
> # 第一级标题
> ## 第二级标题
> ### 第三级标题
> #### 第四级标题
> ##### 第五级标题
> ###### 第六级标题

鉴于第五级标题和第六级标题文字偏小，推荐仅使用前四级或前三级标题。

我们把一本书的书名称为第 0 级标题，一章标题称为第 1 级标题，以此类推。

前两级标题也可以这样表示：

```
因下面的 === 我会成为第一级标题
===
因下面的 --- 我会成为第二级标题
---
```

效果如下：

> # 因下面的 === 我成了第一级标题
> ## 因下面的 --- 我成了第二级标题

扩展，如果`空行`后是 `===` 或 `---`，例如：

```

===

---
```

效果如下：

===

---

可见，`===` 和 `---` 显示效果分别为细分割线和宽分割线。


### 引用（Blockquotes）

使用大于号 `>` 在段首来引用文字。

```
> 这是引用
> 这是另一段引用
```

效果如下：
> 这是引用  
> 这是另一段引用


### 强调（Emphasis）之粗体（Bold）

使用两个星号 `**` 在文字前后来**加粗文字**。

```
**我会变粗**
```

效果如下：

**我已变粗**

备注：
* Markdown Editor 中，如紧接两个星号 `**` 内的文本是`空格`，无法实现加粗。`GitHub` 和有道云笔记亦如是。

### 强调之斜体（Italic）
在文字前后加一个星号 `*` 来让文字倾斜。

```
*我要成比萨斜塔了*
```

效果如下：

*我已经是比萨斜塔了*

备注：
* 侯捷在《Word 排版艺术》明确反对给中文使用斜体，歪歪扭扭的汉字完全丧失了书法美感与方块字的韵律。请不要误会，英文的斜体很美观，侯捷不但没反对而且推荐，比如 *`TypeisBeautiful.com`*，[links](http://www.typeisbeautiful.com/)。
* 说起中文字体和英文字体的差异，有一点值得一提，相同字号/大小、相同行距的中英文混排效果是很差的，原因不解释了。
* 字体不误用，需要注意的一点是不要做什么：不要给英文应用中文字体，如宋体，虽然很多时候这个坑是 Word 默认设置把我们带进去的。

### ~~强调之下划线（Underline）~~
想给一段文字加上下划线，只需要在**文字前后加一个下划线 `_`**。

```
_我是下划线_
```

效果如下：

__我是下划线__（`~~` 不是下划线，而是**加粗**~~）

备注：
* 在 `Markdown Editor` 中，该功能无法实现，
* 待在 `GitHub` 和有道云笔记中尝试一下。→
* 下划线美观性不够。

### 强调之删除线（Strikethrough）
给文字加上删除线效果，只需要在**文字前后加~~一个~~两个波浪线 `~~`**。

```
~~我所说的都是错的~~
```

效果如下：

~~我所说的都是错的~~

备注：
* 波浪线不分中英文。添加图片时，感叹号是分中英文的，其实更准确的表达是分全角/半角。
* 有道云笔记中，如果波浪线 `~~` 紧跟了`空格`，效果无法实现。

### 插入链接（Links）
链接把文字指向网址，是一种快捷方式，也是深入阅读和扩展阅读的方式。格式：中括号和小括号，`[文字](链接)`。

```
[Google](http://google.com)
```

效果如下：  
[Google](http://google.com)

备注：
* 链接中的井号 `#`可实现精确定位。

### 插入图片（Images）

插入图片类似于插入链接，要在前面加上一个感叹号 `!`，注意，感叹号是英文感叹号 `!`，不是中文感叹号！

```
![百度搜索](https://www.baidu.com/img/bd_logo1.png)
```

效果如下：

![百度搜索](https://www.baidu.com/img/bd_logo1.png)

* 如何给图片加链接？



### 列表
列表分无序列表（Unordered, bullet points）和有序列表（Ordered, numbered lists）。

使用减号 `-`、星号 `*` 或加号 `+` 加`空格`来创建无序列表。

```
- 我是无序列表项目一
* 我是无序列表项目二
+ 我是无序列表项目三
```

效果如下：  
- 我是无序列表项目一
* 我是无序列表项目二
+ 我是无序列表项目三

可以构造多层的无序列表，第二层无序列表仅在第一层无序列表前加`空格`即可。

```
* 我是第一层无序列表
  * 我是第二层无序列表
  * 第一层前加一个或两个`空格`可成为第二层无序列表
    * 我是第三层无序列表吗？
    * 在我前面加三个或四个`空格`
      * 在我前面加五个或六个`空格`
```

效果如下：
* 我是第一层无序列表
  * 我是第二层无序列表
  * 第一层前加一个或两个`空格`可成为第二层无序列表
    * 我是第三层无序列表吗？
    * 在我前面加三个或四个`空格`
      * 在我前面加五个或六个`空格`

推荐每增加一级使用两个`空格`。如果每次仅用一个`空格`，会产生问题。

推荐仅使用一层或两层无序列表，如果两级列表无法满足你的要求，需要三层或四层，可以考虑升级列表为标题，以用更大的篇幅作详细的介绍。

备注：
* 有文字的非空行，带星号 `*` 行在文字行文结束按回车，会自动产生下一行星号 `*`，不用再输入星号 `*`
* 如果一行仅有星号 `*`，而没有其他文字，按回车后产生空行，不再有星号 `*`。
* 

使用数字圆点加`空格`如 `1.`、`2.` 来创建有序列表。

```
1. 我是有序列表项目一  
2. 我是有序列表项目二  
3. 我是有序列表项目三 
```

效果如下：

1. 我是有序列表项目一  
2. 我是有序列表项目二  
3. 我是有序列表项目三 

### 其他标记之换行

一段文字结束，只是回车是不能换行的。最终如果有几个段落，显示效果是几段文字会连在一起。

段落和段落之间强制空一行才能分段换行。


方案：
* 段落间空一行
* 一段后空两格【在 `GitHub` 中，段末双`空格`无法实现换行效】
* 

相关：
* 软回车
  * Markdown 好像没有软回车一说

### 其他标记之分割线

在一行内使用三个以上的星号 `***` 或减号 `---` 来创建分割线。

```
***
---
```

效果如下：
***
---

注意，一个例外是，空行的影响，没有空行时：

```
第一段文字
---
第二段文字
***
```

效果如下：
第一段文字
---
第二段文字
***

注意，一个例外是，空行的影响，有空行时：

```
第一段文字

---

第二段文字

***

```

效果如下：

第一段文字

---

第二段文字

***

可以这样表述：空行 + 三个或以上 `---` 或 `***` + 空行，也就是前后有空行，显示效果是没有问题的。

实际上，仅前面有空行，显示效果就没有问题。


备注：
* 行内不能再有其他任何东西。

## 推荐实务
我想在此写下一些具有鲜明态度的写作实务。我先试着在心里酝酿了一张清单，然后在遇到相关的介绍/依据后尽量收集记录下来，随着时间的积累于是就慢慢有了这张大家看得到的清单。
* 中文斜体
* 日期的写法 ISO，2016-09-05。
  * 参见 `GBT 7408-2005 = ISO 8601-2000 ＜2016.04＞ 数据元和交换格式 信息交换 日期和时间表示法`
  * 1.年月日顺序。先年后月再日
  * 2.连接线。连接线是中间横杠 -，不是下横杠_，也不是点 .。手写日期事用点是最常见的
  * 3.日期位数完整。年建议四位小数、月和日两位小数，个位时在是前面补上 `0`，如 `2016-01-01`，而不是 `16-1-1`，后者造成混淆与误会的概率不经意间就增加了很多倍。
* 列表文末的不用句号
* 表格文末的不用句号
* 中英混排时，英文单词和数字前后留空格。对纯中文来说，段内无任何空格。参见苹果主页排版。
