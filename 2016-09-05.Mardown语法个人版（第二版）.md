## Markdown 是什么？

Markdown 既不是工具也不是程序语言，而是一种轻量级的使用纯文本编写的“标记语言”。

Markdown 写作可以直接在 txt 记事本里进行，也可以选择专业的编辑器。后者可以实时预览排版效果，方便点选语法菜单；但 Markdown 的语法相对来说是非常简单的，掌握基本语法后即使在记事本里写作也是完全没有问题的。这样我就摆脱了环境和条件的限制。

## Markdown 基础语法

写作过程中，一般需要这样的排版：
* 标题
* 引用
* 强调（加粗、斜体、下划线、删除线）
* 插入链接、插入图片
* 列表（有序列表、无序列表）
* 表格
* 注释

## 标题

Markdown 可支持 6 个级别的标题（对应于 html 的 h1~h6）。
写法/语法：使用井号 # 加空格在行首来创建标题。几个井号就是几级标题。

```
# 第一级标题
## 第二级标题
### 第三级标题
#### 第四级标题
##### 第五级标题
###### 第六级标题
```

效果如下：
# 第一级标题
## 第二级标题
### 第三级标题
#### 第四级标题
##### 第五级标题
###### 第六级标题

建议：鉴于第五级标题和第六级标题文字偏小，推荐仅使用前四级或前三级标题。  


## 引用

使用大于号 > 在段首来引用一段文字。

```
> 这是引用
> 这是另一段引用
```

效果如下：
> 这是引用  
> 这是另一段引用


## 强调之粗体

在文字前后加两个 * 号来加粗文字。

```
**我会变粗**
```

效果如下：

**这是我变粗后的样子**

## 强调之斜体
让文字倾斜，只需在文字前后加一个 * 号。

```
*我要成比萨斜塔了*
```

效果如下：

*我已经是比萨斜塔了*

备注：
* 侯捷在《Word 排版艺术》反对给中文使用斜体，歪歪扭扭的汉子没有任何书法之美。但有一点不要误会，英文的斜体很美观，侯捷不但不会反对并有推荐。
* 说起中文和英文字体的不同，有一种情况是，相同字号/大小、相同行距的中英文混排效果是很差的，原因不解释了。
* 字体不误用，需要注意的一点是不要做什么：不要给英文应用中文字体，如宋体；虽然这个坑很多时候是 Word 默认设置把我们带进去的。

## 插入链接
链接把文字指向网址，是一种快捷方式，也是深入阅读和扩展阅读的方式。格式：中括号和小括号，[文字](链接)

```
[Google](http://google.com)
```

效果如下：  
[Google](http://google.com)

备注：
* 维基百科实现三大基本要求的 xxx 主要靠注释来源，内文打开方便主要使用“插入链接”，其他方式有：索引表，xxx。

## 列表
列表分无序列表和有序列表。

使用减号 -、星号 * 或加号 + 加空格来创建无序列表。

```
- 我是无序列表项目一
* 我是无序列表项目二
+ 我是无序列表项目三
```

效果如下：  
- 我是无序列表项目一
* 我是无序列表项目二
+ 我是无序列表项目三

使用数字圆点加空格如 “1.”、“2.” 来创建有序列表。

```
1. 我是有序列表项目一  
2. 我是有序列表项目二  
3. 我是有序列表项目三 
```

效果如下：

1. 我是有序列表项目一  
2. 我是有序列表项目二  
3. 我是有序列表项目三

## 其他标记之分割线

在一行内使用三个以上的星号 *** 或减号 --- 来创建分割线。

```
***
---
```

***
---

## 推荐实务
我想在此写下一些具有鲜明态度的写作实务。我先试着在心里酝酿了一张清单，然后在遇到相关的介绍/依据后尽量收集记录下来，随着时间的积累于是就慢慢有了这张大家看得到的清单。
* 中文斜体
* 列表文末的句号
* 日期的写法 ISO，2016-09-05。
  * 第一点，年月日顺序。先年后月再日
  * 第二点，连接线。连接线是中间横杠 -，不是下横杠_，也不是点 .。手写日期事用点是最常见的
  * 第三，日期位数完整。年建议四位小数、月和日两位小数，个位时在是前面补上 0，如 2016-01-01，而不是 16-01-01，后者导致混淆与误会的概率不经意间就增加了很多倍。
* 表格文末的标点/句号
* 中英混排时，英文单词和数字前后留空格。对纯中文来说，段内无任何空格。参见苹果主页文案排版。
